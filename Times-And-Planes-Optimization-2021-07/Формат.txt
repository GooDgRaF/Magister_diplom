Файл с данными точек.
В первой строке: Единицы измерения для каждой позиции (m, km, NM, m_s, km_h, NM_h)
 
Со второй перечисляются все точки (каждая следующая с новой строки) в формате:  
Название x y z Vmin Vmax флаг_посадочной_полосы 
Название начинается с буквы.
    x,y,z из R;
    Vmin,Vmax из R+;
    флаг_посадочной_полосы - LAND иначе пусто или ноль.

Пример:
3
m km NM km_h NM_h
P1 0 0 0 10 20 LAND
P2 1 1 1 10 22 0
P3 3 3 3 10 24
--------------------------------------------------------------------------------------------------
Описание спрямлений (считаем, что зоны разных спрямлений не перекрываются):
Str(Название1 Название2  ...  НазваниеН) - Возможность прервать выполнение текущей схемы и уйти на заданную(ые) точки.

Действует на сегменты с концами в точках, заключенных между Str(...) Точка1 Точка2 ... ТочкаК /Str

--------------------------------------------------------------------------------------------------
Файл с данными о схемах:

 -Считаем, что схемы пересекаются только по началу и концу
 -В одной схеме может быть только одно спрямление
 -В данном файле описываются схемы - связи между точками. И зоны ожидания (ЗО).
 -Конечными точками схемы считаются ее последняя точка, а также не принадлежащие ей точки, на которые возможно спрямление.


В первой строке: количество схем - число из N
Во второй строке: количество ЗО - число из N

Начиная с третьей строки перечисляются все схемы (каждая следующая с новой строки) в формате:
Название схемы (точка начала схемы)(точки конца схемы) : название точек через пробел, следуя с начала [спрямление [название точек через пробел]] название точек через пробел
Название схемы - любой набор букв и цифр
Названия точек - начинаются с буквы



Некоторый "синтаксический сахар":
1)Если схема состоит только из двух точек: 'NameTwoP (A)(B): A B'
Можно записать как: 'NameTwoP (A)(B):'
--------
В этом же файле информация о зонах ожидания: 
В каждой строке одна ЗО. t_min, t_max из R+. MU - единицы измерения (s, min, h)
Название ЗО (точка начала ЗО): t_minMU t_maxMU 

Пример схем:
NameA (DIPOP)(EE500 KOLOS): Str(KOLOS) DIPOP EE500 /Str
NameB (KOLOS)(BEKAR) : KOLOS VALET Str(RODEL) EE020 EE021 EE022 EE023 EE024 EE025 /Str RODEL BEKAR
NameС (RODEL)(RW25R): RODEL BEKAR KVOTA EE252 EE253 RW25R
Name_waiting_area (GALEB): 2min 0.3h

Пример файла схем:
1
1
Name1 (P3)(P1): Str(P1) P3 P2 /Str P1
WA1 (P3): 180s 600s

Файл с данными потоков:
В первой строке: количество потоков - число из N
Со второй строки (каждый новый поток в новой строчке) потоки в формате:
Имя потока    Имя первой точки этого потока

Пример:
Flow1 DIPOP
